# نَسَّق | NASSAQ - Product Requirements Document

## نظرة عامة
نظام إدارة مدارس متكامل يعمل بالذكاء الاصطناعي، متعدد المستأجرين (Multi-tenant)

## المستخدمون
1. **مدير المنصة (Platform Admin)** - إدارة جميع المدارس والمستخدمين
2. **مدير المدرسة (School Principal)** - إدارة مدرسة واحدة
3. **المعلم (Teacher)** - إدارة الفصول والحصص
4. **الطالب (Student)** - عرض الجدول والدرجات
5. **ولي الأمر (Parent)** - متابعة أداء الأبناء

## الميزات المُنجزة

### مركز القيادة (Command Center) ✅ (محدّث مارس 2026)
- أزرار موحدة: "طالب / طلاب"، "معلم / معلمين"، "فصل / فصول"
- كروت مؤشرات الأداء (الطلاب، المعلمين، الفصول) قابلة للنقر
- قسم "يحتاج تدخل الآن" للمشكلات العاجلة
- نسبة الحضور اليوم (المعلمين والطلاب)
- **جميع الكروت تعرض بيانات ديناميكية من قاعدة البيانات فقط**
- **كارت المعلمين يعرض الرقم الصحيح (10) من قاعدة البيانات**
- **API جديدة: `/api/admin/command-center/stats`**

### تنسيق رأس الصفحات الموحد ✅ (مارس 2026)
- اسم المستخدم يظهر أولاً وبخط أكبر (text-2xl)
- عنوان الصفحة يظهر تحت الاسم (text-base)
- تم تطبيقه على: مركز القيادة، إعدادات المدرسة، إدارة المستخدمين، الجدول المدرسي

### الجدول المدرسي ✅
- زر "معالجة الجدول بالذكاء الاصطناعي" (بدلاً من "توليد")
- إزالة زر "+جديد"
- تكامل مع إعدادات المدرسة

### إدارة المستخدمين والفصول ✅ (محدّث مارس 2026)
- أزرار موحدة مع مركز القيادة
- فلتر تلقائي عند النقر من الكروت
- **عرض ديناميكي للبيانات من قاعدة البيانات فقط**
- **اسم المستخدم يظهر أولاً في رأس الصفحة**

### إدارة حضور المعلمين والاداريين ✅
- تغيير الاسم من "إدارة الحضور"
- تقارير نسب الحضور الفعلية

### مركز التواصل والاشعارات ✅

### صفحة إعدادات المدرسة - مصدر الحقيقة الوحيد ✅ (محدّث مارس 2026)

**المبدأ الأساسي:** Single Source of Truth - صفحة إعدادات المدرسة هي المصدر الرسمي الوحيد لبيانات المدرسة

**إعادة الهيكلة الكاملة (مارس 2026):**
تم تقسيم الصفحة إلى قسمين رئيسيين:

#### القسم الأول: البيانات المتغيرة لبناء الجدول (Dynamic Timetable Inputs)
هذا القسم يحتوي على المدخلات التشغيلية لمحرك الجدول:
- **العام والفصل الدراسي** - اختيار العام والفصل الدراسي الحالي
- **أيام العمل** - تحديد أيام الدراسة والعطلة (7 أيام تفاعلية)
- **التوقيت والحصص** - وقت بداية اليوم، عدد الحصص، مدة الحصة
- **الاستراحات والصلاة** - فترات الاستراحة وصلاة الظهر
- **الفصول والشعب** - إدارة الفصول الدراسية
- **إسناد المعلمين** - ربط المعلمين بالمواد
- **عدم التوفر** - قيود توفر المعلمين والفصول
- **القيود والتفضيلات** - القيود الإلزامية (Hard) والتفضيلية (Soft)

#### القسم الثاني: البيانات الأساسية الثابتة العامة (Official Static Reference Data)
بيانات مرجعية رسمية للقراءة فقط:
- **المنهج الرسمي** - إحصائيات المنهج الدراسي من وزارة التعليم
- **المراحل والمسارات** - المراحل الدراسية (3) والمسارات التعليمية (8)
- **النصاب التعليمي** - نصاب كل رتبة معلم (حصص/أسبوع)
- **توزيع المواد** - توزيع المواد لكل صف

#### بطاقة جاهزية الجدول (Readiness Card) ✅
- **عنوان محدث:** "بيانات الجدول غير مكتملة" / "بيانات الجدول جاهزة جزئياً" / "جاهز لإنشاء الجدول!"
- **عرض النسبة المئوية** في دائرة تقدم
- **عرض البيانات المطلوبة** في كروت ملونة مع:
  - اسم الفئة (السياق الأكاديمي، أيام الدراسة، هيكل اليوم، إلخ)
  - النقاط المكتسبة/الإجمالية
  - قائمة بالمشاكل الحرجة
  - **زر "إصلاح"** للانتقال المباشر للتبويب المناسب
- **عرض البيانات المكتملة** بشارات خضراء
- **تحديث تلقائي للنسبة** بعد حفظ أي تعديلات

**الإصلاحات الحرجة المُنفذة:**
1. ✅ **CRUD المواد الدراسية:**
   - جلب المواد من `/school/subjects` بدلاً من `/reference/subjects`
   - تعديل وحذف المواد يعمل بشكل صحيح
   
2. ✅ **CRUD القيود الإدارية:**
   - إصلاح: POST كان يحفظ في `admin_constraints` بينما PUT/DELETE يبحثان في `school_constraints`
   - الآن جميع العمليات تستخدم `school_constraints` موحداً
   
3. ✅ **تعديل الأوقات المباشر (Inline Edit):**
   - مكون جديد `TimeSlotInlineEdit` للتعديل المباشر
   - إعادة حساب تلقائي متسلسل للفترات اللاحقة
   - حفظ فوري في قاعدة البيانات عند الـ blur أو Enter
   - زر "إعادة الضبط" للوضع الافتراضي
   
4. ✅ **تصميم بطاقة أيام العمل:**
   - كروت صغيرة لكل يوم من الأسبوع
   - ألوان مميزة (أخضر للدراسة، رمادي للعطلة)
   - دوائر إحصائية للملخص

**APIs المستخدمة:**
- `GET /school/settings` - جلب جميع الإعدادات
- `PUT /school/settings` - تحديث الإعدادات
- `GET/POST/PUT/DELETE /school/subjects` - إدارة المواد
- `GET/POST/PUT/DELETE /school/constraints` - إدارة القيود
- `GET /school/settings/stages` - المراحل التعليمية
- `GET /school/settings/audit-logs` - سجل التعديلات
- `GET /api/official-curriculum/stats` - إحصائيات المنهج الرسمي
- `GET /api/official-curriculum/stages` - المراحل الدراسية
- `GET /api/official-curriculum/tracks` - المسارات التعليمية
- `GET /api/official-curriculum/teacher-rank-loads` - نصاب المعلمين
- `GET /api/timetable-readiness/check` - فحص جاهزية الجدول

### تثبيت الإعدادات الأساسية والهيكل الأكاديمي ✅ (مارس 2026)

- أيام العمل: الأحد - الخميس

- التوقيت: 07:00 - 13:15
- الفترات الزمنية: 9 فترات (7 حصص + استراحة + صلاة)
- مدة الحصة: 45 دقيقة

8. لا يمكن إسناد مادة إلى معلم غير مؤهل لها
9. يجب احترام عدد الحصص الأسبوعية المطلوبة لكل مادة
10. لا يمكن وضع حصة في فترة غير متاحة للمعلم أو الفصل
11. لا يمكن استخدام فترات زمنية معطلة أو مغلقة
12. يجب احترام متطلبات نوع الغرفة (مختبر، حاسب...)
